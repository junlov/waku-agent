# Lab decision

## Context

Name the observed failure, affected product promise, workload boundary, and the
evidence that makes this a real decision.

## Options

Record two or three viable approaches. For each, state the correctness model,
operational cost, failure mode, and what changes at the next chapter's scale.

## Decision

State the chosen approach and the invariant it protects. Separate measured facts
from assumptions and name any authority boundary that must remain outside model
input.

## Proof and reversal

Name the deterministic check, realistic user path, rollback condition, and the
evidence that would cause this decision to be revisited.
